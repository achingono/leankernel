# Docs Style Guide

LeanKernel documentation uses implementation-first conventions.

## Required Structure

- Use kebab-case file names.
- Keep docs in hierarchical folders by domain.
- Ensure every docs folder has an `index.md`.

## Authoring Rules

- Prefer one canonical page per topic.
- Link related docs rather than duplicating long sections.
- Include source references to code paths for behavior claims.
- Keep roadmap/future work under `docs/plans/` only.

## Link and Markdown Hygiene

- Use repository-relative links.
- Add language tags to fenced code blocks.
- Keep headings consistent and concise.
