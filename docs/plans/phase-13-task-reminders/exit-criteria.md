# Phase 13 Exit Criteria

## Gate Checklist
- [ ] Users can capture tasks/reminders explicitly from any channel with correct NL due parsing.
- [ ] Inferred follow-ups are captured only after user confirmation.
- [ ] Full lifecycle (create/update/complete/snooze/cancel/list) works and is person-scoped.
- [ ] Recurrence and due firing run via the scheduler; overdue items escalate.
- [ ] Proactive reminders deliver on the preferred/available channel, respecting quiet hours.
- [ ] Users can complete or snooze a reminder from the delivery channel.
- [ ] Tasks are visible and actionable across all of the person's channels.
- [ ] Unit + integration tests cover capture, parsing, recurrence, delivery, and acknowledgement.
