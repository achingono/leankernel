# Phase 11 Exit Criteria

## Gate Checklist
- [ ] A user can authorize a provider via OAuth (or API key) and see it listed as connected.
- [ ] Credentials are stored encrypted at rest, person-scoped, and tenant-isolated.
- [ ] Tokens refresh automatically; expiry and insufficient-scope trigger re-consent.
- [ ] Provider operations are exposed as governed tools honoring governance and egress rules.
- [ ] Authorizations follow the person across channels (via Phase 10 identity).
- [ ] Disconnect/revoke clears local vault entries and upstream grants where supported.
- [ ] A reference connector performs an authorized call end to end.
- [ ] Unit + integration tests cover flow, refresh, encryption, revocation, and isolation.
