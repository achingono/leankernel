# Phase 09 Exit Criteria

## Gate Checklist
- [ ] Blazor app shell runs in the gateway with routing, navigation, and session continuity.
- [ ] Authenticated users can chat with full session/history/turn state.
- [ ] Users can search, browse, and edit knowledge and view document library/ingestion status.
- [ ] Onboarding wizard guides identity setup and reflects detected gaps.
- [ ] Operators can inspect per-turn diagnostics via the diagnostics explorer.
- [ ] Admin console shows provider health/routing/spend and controls tool governance and scheduler.
- [ ] Every surface is partition-aware with no cross-tenant/user leakage.
- [ ] Component + Playwright tests cover chat, knowledge, onboarding, diagnostics, and admin flows.
