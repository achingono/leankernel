# Phase 12 Exit Criteria

## Gate Checklist
- [ ] The assistant can search, read, and summarize the user's email threads.
- [ ] The assistant can triage/label and draft replies; sending requires approval (draft-only fallback if the engine is absent).
- [ ] The assistant can read availability across calendars with correct timezone handling.
- [ ] The assistant can create/update/cancel events and send/respond to invites, gated by approval.
- [ ] Email/calendar context and derived facts are person-scoped and available across channels.
- [ ] Optional daily briefings deliver via scheduler + channels when enabled.
- [ ] The provider adapter is swappable behind provider-agnostic contracts.
- [ ] Unit + integration tests cover triage, gating, availability, event lifecycle, and cross-channel context.
