# Phase 05 Exit Criteria

## Gate Checklist
- [ ] Filesystem tools operate only within the configured root; traversal/symlink abuse is rejected.
- [ ] Database query is read-only by default, parameterized, and safe against injection.
- [ ] JSON transform and CSV/XLSX read/write produce correct, tested results.
- [ ] `web_fetch`/`http_request` honor egress validation and secret handling.
- [ ] Browser tool works when the automation service is up and degrades cleanly when down.
- [ ] Monitored-folder documents are ingested idempotently into the knowledge store with dedupe.
- [ ] All new tools are registered through `IToolRegistry` and constrained by governance.
- [ ] Unit + integration tests cover boundaries, injection, egress, degradation, and ingestion.
